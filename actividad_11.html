Actividad 1.1

SIMULACIÓN DE PROYECTO IoT EN PAREJA

Esta tarea consiste en la simulación de un proyecto IoT. Para ello haremos dos grupos:

          A) Se encargará de la fase de identificación de necesidades y diseño

          B) Se encargará de las fases de implementación, pruebas y validación, y despliegue y mantenimiento

PROYECTO IoT

PROYECTO EPIS EMPRESA "X"

Una empresa internacional de automoción necesita organizar de forma óptima los EPIs (Equipos de Seguridad Individual) necesarios para la fase de producción de los automóviles y ha contratado a la empresa “X” para que gestione el proyecto IoT. Para ello, se desarrollará un sistema de dispensadores de EPIs que funcionen a demanda de los operarios.
El funcionamiento será el siguiente: cuando un operario necesite un EPI, se dirigirá a la máquina dispensadora y, con su tarjeta identificativa, podrá retirar el producto deseado. Esta retirada quedará registrada en un software de gestión que contará la adquisición como una venta, descontándola del stock de la máquina. Cuando el stock del artículo descienda a la mitad, se enviará una notificación a la empresa “X” para que reponga el material.
Es crucial que la empresa “X” gestione con precisión el inventario y las reposiciones, ya que, de no ser así, la producción podría detenerse, llevando el proceso al fracaso.


TAREAS:

GRUPO A

  1. Identificación de necesidades: definir brevemente el problema o necesidad que el proyecto IoT resolverá.
  2. Diseño: esbozar un diagrama simple del sistema incluyendo los componentes principales.

GRUPO B

  1. Implementación: hacer una lista rápida de los pasos necesarios para construir el prototipo, así como los recursos necesarios.
  2. Pruebas y validación: definir brevemente cómo se probará el sistema y qué criterios determinarán si el sistema es exitoso.
  3. Despliegue y mantenimiento: planificar cómo se instalará el sistema en el entorno real y qué mantenimiento será necesario para garantizar el funcionamiento continuo.

PRESENTACIÓN Y DISCUSIÓN

Cada estudiante/grupo presentará su parte del proyecto y posteriormente se discutirá cómo las fases interactúan entre sí y la importancia de cada una en el éxito del proyecto IoT.
